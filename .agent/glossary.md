# Plugin Glossary

## Primary Objective
Build and maintain a WordPress payment gateway plugin that integrates SSLCommerz with Easy Digital Downloads (EDD).

## Canonical Documentation Sources
- SSLCommerz v4 API docs: https://developer.sslcommerz.com/doc/v4/
- Easy Digital Downloads API reference: https://easydigitaldownloads.com/categories/docs/api-reference/

## Core Entities
- `payment_id`: EDD internal payment post ID.
- `tran_id`: Merchant-side transaction ID generated by plugin (`EDD-{payment_id}-{timestamp}`).
- `val_id`: SSLCommerz validation identifier returned after payment attempt.
- `bank_tran_id`: Acquirer/bank transaction reference from SSLCommerz.
- `refund_ref_id`: SSLCommerz refund reference identifier.

## Gateway Modes
- `hosted`: Customer is redirected to SSLCommerz gateway page.
- `popup`: EasyCheckout flow using `embed.min.js` and AJAX init endpoint.

## Callback Channels
- `ipn`: Server-to-server callback to plugin IPN URL.
- `return`: Customer browser redirect/callback (`edd-listener=sslcommerz_return`).

## Query Keys / Params
- `edd-listener`: Router-like query key for callback handling.
- `edd-sslcz-pid`: Payment ID passed in return URLs.
- `status`: Return status (`success`, `fail`, `cancel`).

## Stored Payment Meta Keys
- `_sslcommerz_tran_id`
- `_sslcommerz_val_id`
- `_sslcommerz_bank_tran_id`
- `_sslcommerz_card_type`
- `_sslcommerz_amount`
- `_sslcommerz_currency`
- `_sslcommerz_refund_ref_id`

## Important Plugin Functions
- `edd_sslcommerz_process_payment`: Hosted checkout payment initialization.
- `edd_sslcommerz_ajax_init_popup`: Popup mode payment initialization via AJAX.
- `edd_sslcommerz_return_handler`: Customer redirect callback validator.
- `EDD_SSLCommerz_IPN_Handler::handle`: IPN validation and status update handler.
- `edd_sslcommerz_get_option`: Gateway settings accessor.

## External APIs
- Session init: `/gwprocess/v4/api.php`
- Validation: `/validator/api/validationserverAPI.php`
- Refund init: `/validator/api/merchantTransIDvalidationAPI.php`
